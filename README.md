# Neon Breaker（项目代号）

一款使用 `Vue 3 + Pinia + Vite + Canvas 2D` 开发的纯前端竖屏霓虹打砖块游戏。项目持续按阶段迭代，最终目标包含 20 个主线关卡、4 个 Boss、5 种即时道具、金币与永久升级、无尽模式，以及电脑和手机双端操作。

当前版本：`v0.6.0` 第一章与首个 Boss。正式游戏名称将在后续确定。

当前里程碑进度：`6 / 9` 个开发阶段完成，约 `67%`。

## 当前可玩内容

- `540 × 960` 竖屏霓虹战场
- 经典挡板、小球和砖块反弹玩法
- 第一章第 1～5 关全部使用独立手工砖阵、说明、目标、奖励和递增球速
- 一击破碎的普通砖
- 需要两次命中的紫色强化砖
- 3 条生命、掉球重置、胜利与失败
- 键盘、鼠标和触摸控制
- 全量冻结暂停、页面失焦自动暂停、3 秒恢复倒计时、重新挑战和全屏
- 基础粒子、闪光、拖尾、辉光和震屏
- 命中停顿、冲击波、浮动分数、强化粒子池和连击反馈
- 多球分裂、挡板扩展、穿透光球、时流减速和双轨激光
- 最多 12 颗球，不同道具同时生效，同类计时道具刷新时间
- 晶币散落、重力弹跳、挡板磁吸和本地持有数量
- 发光胶囊掉落、挡板接取、漏接消失和掉球统一清空
- 连击倒计时、分数倍率和最高连击
- 标题页、任务简报、准备发球、游戏、暂停、胜利和失败的完整流程
- 四章 20 关战役星图、顺序锁定、关卡预览和数据驱动砖阵
- 第 5、10、15、20 关 Boss 节点框架
- 第 5 关“棱镜核心”正式 Boss：移动核心、三阶段护盾阵列、12 点核心耐久和阶段脉冲
- Boss 护盾/核心碰撞、激光伤害、阶段重构、专属 Canvas HUD、桌面终端与手机状态条
- 每关独立名称、主题色、目标分数、目标连击、球速和清关奖励
- 第一关三星评价：通关、剩余生命、目标分数或连击
- 清关固定奖励、正式胜负结算和历史记录不降级
- `gamename:save` 版本化正式存档
- v0.3 Pinia 快照自动迁移、原始备份和损坏存档恢复
- 关卡星级、最高分、最佳连击、挑战次数、通关次数和最佳剩余生命记录
- 桌面与手机均支持两步确认重置本地进度
- 8 档累计星级奖励，领取状态进入正式存档
- 7 类永久强化：挡板宽度、道具概率、晶币倍率、磁吸范围、连击窗口、能量护盾和额外生命
- 强化购买、满级/晶币不足状态、两步免费重置与全额退款
- 7 类强化全部真实装载到下一局运行参数
- 桌面三栏 UI 和手机单屏 UI
- Pinia 状态摘要与最高分本地保存

当前版本的 20 关均可由数据框架装载并进行战斗，其中第 1～5 关已经完成正式手工设计，第 5 关具备完整三阶段 Boss 机制。第 6～20 关最终设计、其余 3 个 Boss 和无尽模式将在后续阶段继续完成。

## 操作说明

### 电脑

- 鼠标移动：控制挡板
- `A` / `D`：控制挡板
- `←` / `→`：控制挡板
- 点击战场或 `Space`：发球
- 激光生效时点击战场或 `Space`：发射双轨激光
- `P` 或 `Esc`：暂停；再次按下启动 3 秒恢复倒计时
- `F`：切换全屏

### 手机

- 在战场内左右拖动：控制挡板
- 轻触战场或底部按钮：发球
- 页面底部按钮：暂停或启动 3 秒恢复倒计时
- 切换后台或页面失焦：自动暂停
- 激光激活时，发射与暂停使用两个独立按钮

### 战役与强化

- 从标题进入“战役”，按章节选择已解锁关卡。
- 点击锁定关卡可以预览目标，但必须顺序通关才能部署。
- “强化”页面购买永久模块，升级在下一局装载。
- 强化页面可以免费重置，全部投入晶币会原额返还。
- 达到星级门槛后，在战役页点击对应奖励领取晶币。

### 棱镜核心 Boss

- 每阶段先击碎当前护盾砖阵，护盾破裂后才能伤害移动核心。
- 核心每损失 4 点耐久进入下一阶段并重构护盾，共 3 个阶段、12 点耐久。
- Canvas 顶部、桌面操作终端和手机状态栏会同步显示阶段、护盾节点和核心耐久。
- 第三阶段核心移动最快；击破最终核心后才会结算胜利并解锁第 6 关。

## 技术栈

- Vue 3：界面、HUD 和操作按钮
- Pinia：生命、得分、关卡进度和设置摘要
- Canvas 2D：游戏实体、碰撞、砖块和特效
- Vite：开发服务器和生产构建
- localStorage：最高分与后续正式存档
- pnpm：依赖管理

## 环境要求

- Node.js 20.19+ 或 22.12+
- pnpm 10+
- 支持 Canvas 2D 的现代浏览器

## 测试接口

- `window.render_game_to_text()`：读取页面模式、运行编号、三星结算、正式存档摘要、生命、得分、晶币、连击、挡板、小球、砖块、掉落物、道具和特效状态。
- `window.advanceTime(ms)`：按固定时间步确定性推进游戏。
- 开发环境提供 `window.__NEON_BREAKER_TEST__`，用于胜负和物理边界回归。
- 开发环境提供 `window.__NEON_BREAKER_SAVE_TEST__`，用于正式存档、迁移、恢复和重置回归。
- 开发环境提供 `window.__NEON_BREAKER_PROGRESSION_TEST__`，用于 20 关装载、升级购买/退款和星级奖励回归。

## 存档说明

当前 `v0.6.0` 继续使用 `saveVersion = 2` 的 `gamename:save` 正式存档，保存晶币、20 关记录、解锁进度、星级奖励领取、7 类永久升级、无尽占位和设置。本阶段未改变存档结构；v1 正式存档和更早的 `gamename:game` 快照仍会自动迁移，解析失败时先备份原始字符串再建立安全新档。
