# 版本变更记录

本项目采用语义化版本号。每个开发阶段完成浏览器验证后，都会提交代码、创建版本标签并推送远程仓库。

## [0.5.0] - 2026-07-20

### 新增

- 新增 4 章 20 关数据模型，每关包含名称、章节、说明、主题色、三星目标、球速、奖励和确定性砖阵。
- 新增第 5、10、15、20 关 Boss 节点框架，为后续专属 Boss 机制预留数据入口。
- 新增完整战役星图：章节切换、5 关节点、锁定预览、历史星级、章节进度和关卡部署。
- 新增 3/6/9/12/15/30/45/60 星八档累计奖励，并持久化领取状态。
- 新增强化实验室与 7 类永久模块：挡板宽度、胶囊概率、晶币倍率、磁吸范围、连击窗口、能量护盾和备用光核。
- 新增升级成本、等级条、满级/晶币不足状态、购买反馈和两步免费重置，重置时全额返还投入晶币。
- 新增 7 类升级的运行参数装载；护盾可以抵挡掉球，额外生命同步扩展 Canvas 与外部 HUD。
- 新增球路 7 秒无命中自动微调机制，避免规则砖阵形成长时间循环。
- 正式存档提升至 `saveVersion = 2`，新增星级奖励领取记录和 v1→v2 自动迁移标记。
- 新增战役与强化专用文本测试状态和 `__NEON_BREAKER_PROGRESSION_TEST__` 调试接口。
- Node 单元测试扩充至 14 项，覆盖 20 关建模、升级参数、购买、退款、锁定、奖励和键盘输入隔离。
- 新增 [v0.5.0 验收报告](./docs/v0.5.0验收报告.md)。

### 调整

- 标题主操作由“进入任务”升级为“进入战役”，完整流程变为标题 → 战役 → 选关 → 简报 → 战斗 → 结算。
- GameEngine 从固定第一关改为接收关卡配置与永久强化参数。
- 胜负结算根据当前关卡写入对应记录，并按顺序解锁下一关。
- 战役与强化页面分别提供 1440×900 完整布局、1280×720 紧凑布局及 390×844/360×800 横向卡片布局。
- 手机战役页压缩至单屏，章节、关卡、详情、奖励与双操作均完整可见。
- 窄屏累计奖励轨改为横向滚动，8 档奖励全部可见、可领取。
- 游戏快捷键仅在标题与战斗页面启用，并跳过按钮、链接和表单控件，确保战役与强化页面可完整使用键盘操作。
- 存档归一化与运行参数均按强化上限截断异常等级；手机顶部导航触控高度提升至 40px。

### 验证

- 20 个关卡配置全部可进入 `briefing`，砖块数和目标有效，Boss 节点严格为 5/10/15/20。
- 使用实际固定时间步、自动挡板和完整碰撞逐关测试，20/20 关均清空砖块进入胜利。
- 七类满级强化实测：挡板 150.8、5 条生命、2 层护盾、3.5 秒连击窗口、190.4 磁吸范围，并验证胶囊概率与晶币倍率。
- 购买 7 项一级强化消耗 350 晶币，两步重置后完整返还 350；刷新后已购等级保持。
- 3 星奖励只能领取一次，领取后增加 30 晶币并持久化。
- 1440×900、1280×720、390×844 和 360×800 均无横向溢出；主操作和底部功能完整可见。
- `pnpm test` 14/14 通过，生产构建通过，页面无运行错误。

## [0.4.0] - 2026-07-20

### 新增

- 新增标题页、第一关任务简报、准备发球、游戏、暂停、恢复倒计时、胜利和失败的完整状态纵切流程。
- 新增第一关三星目标：完成关卡、至少剩余 2 条生命、达到 15000 分或 35 连击。
- 新增清关固定 20 晶币奖励、正式胜负结算、运行编号和本局晶币统计。
- 新增 Canvas 三星结算卡、最终得分、最高连击、本局晶币和剩余生命展示。
- 新增桌面任务历史面板、任务简报面板、结算面板和存档状态卡。
- 新增手机结算条、三星状态、返回标题和条件式战场缩放。
- 新增 `gamename:save` 正式版本化存档服务。
- 新增档案、晶币、主线记录、无尽记录、永久升级占位和设置的分层存档结构。
- 新增关卡星级、最高分、最佳连击、挑战次数、通关次数、最佳剩余生命和最后游玩时间。
- 新增旧 `gamename:game` 快照自动迁移，并保留迁移前原始备份。
- 新增损坏存档原文备份与安全恢复。
- 新增胜负结算幂等保护，避免重复发布状态时重复增加奖励和挑战次数。
- 新增两步确认重置进度和开发环境存档测试接口。
- 新增页面失焦、切换后台自动暂停和 3 秒恢复倒计时。
- 新增手机端紧凑存档状态与两步重置入口。
- 新增 4 项存档服务 Node 单元测试。
- 新增 [v0.4.0 验收报告](./docs/v0.4.0验收报告.md)。

### 调整

- 主线运行的最高连击改为每局重置，历史最佳由正式关卡记录维护。
- 胜利进入正式结算时清除活动球、掉落物、弹体和即时模块。
- 小型桌面隐藏非必要页脚，手机动态状态按需缩小战场，确保主操作保留在首屏。
- 暂停态改为全量冻结物理、挡板、计时道具、掉落物与视觉特效时间。
- 手机激光状态将“发射激光”和“暂停游戏”拆为独立按钮。

### 验证

- 标题 → 简报 → 准备 → 游戏 → 暂停/恢复 → 结算 → 重试/返回标题全链路通过。
- 真实物理自动通关用时 3739 帧（31.2 秒），得到 22727 分、60 连击、61 晶币和三星。
- 新存档、胜利、失败、刷新恢复、旧档迁移、损坏恢复和两步重置全部通过。
- 三星、最高分和最佳连击在较低成绩后不会降级。
- 1440×900、1280×720、390×844 和 360×800 均无横向溢出，主操作完整可见。
- 用户当前 Chrome 完成桌面真实点击以及 390×844、360×800 竖屏复验。
- 自动回归确认暂停 1 秒后挡板位置、粒子数量和道具时间完全不变；倒计时结束后恢复运行。
- 手机 390×844 与 360×800 均可看到存档重置；激光状态仍保留暂停按钮。

## [0.3.0] - 2026-07-20

### 新增

- 新增多球分裂、挡板扩展、穿透光球、时流减速、双轨激光 5 种正面道具。
- 新增最多 12 球的多球循环；计时道具支持同时生效、同类刷新和扩展挡板两层叠加。
- 新增发光胶囊掉落、挡板接取、漏接消失，以及掉球时统一清空临时效果。
- 新增晶币散落、重力、一次弹跳、近挡板磁吸、接取计数与本地持久化。
- 新增双轨激光主动射击和 240ms 射击冷却。
- 新增 2.4 秒连击窗口、分数倍率、最高连击和 HUD 连击进度。
- 新增 12/25ms 命中停顿、扩散波纹、浮动分数、屏幕闪光和增强震屏。
- 新增粒子对象池；高特效上限 600、低特效上限 240。
- 桌面端新增晶币、连击、活动模块列表与实时倒计时。
- 手机端新增晶币、连击、活动模块胶囊和激光主操作按钮。
- 扩充文本测试状态，包含掉落物、弹体、活动道具、晶币、连击和粒子池。
- 新增可重复调用的道具、掉落物、掉球和清关调试入口。
- 新增 [v0.3.0 验收报告](./docs/v0.3.0验收报告.md)。

### 修复

- 多球逻辑由单球索引改为完整球列表，最后一球落底时才扣除生命。
- 修复多个计时道具同时激活时，390×844 和 360×800 底部按钮被挤出视口的问题。
- 激光激活时保留独立暂停入口，避免主按钮职责冲突。

### 验证

- 生产构建通过，浏览器控制台无错误。
- 5 种道具、12 球上限、两层挡板扩展、计时刷新与掉球清空通过。
- 减速位移倍率实测 0.72；穿透球击穿后纵向速度不反转。
- 双轨激光单次发射生成 2 条弹体，并命中 2 点砖块耐久。
- 晶币磁吸接取、连击递增、奖励负载和粒子上限通过。
- 自动挡板在 3739 帧、31.2 秒内清空 51 块砖，3 条生命通关，获得 41 晶币。
- Chrome 桌面实际开始/发球通过；390×844 与 360×800 竖屏 UI 通过。
- 4 个计时道具同时显示时，两档手机视口主操作按钮完全可见且无横向溢出。
