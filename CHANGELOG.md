# 版本变更记录

本项目采用语义化版本号。每个开发阶段完成浏览器验证后，都会提交代码、创建版本标签并推送远程仓库。

## [0.9.0] - 2026-07-20

### 新增

- 新增自动/高/中/低四种画面选择；实际运行使用 HIGH 600、MEDIUM 360、LOW 180 粒子上限，并分别限制拖尾、冲击波和浮分数量。
- 新增“系统与画面”页面，实时展示解析档位、瞬态对象预算、设备线程/内存、系统动态偏好和存档通道。
- 新增画面震动与减弱闪光开关；减弱闪光把全屏白闪限制在 12% 以内，关闭震屏后不再绘制位移反馈。
- 新增自动档能力判断，根据减少动态效果偏好、视口宽度、硬件线程和设备内存选择运行档位。
- 正式存档升级到 v3，新增 `reducedFlash` 和 `auto/medium` 画面值；v1/v2 自动迁移并保留已有进度与设置。
- 新增存储失败降级：localStorage 获取、读写被拒绝时切换会话内存并在设置页显示警告。
- 新增 `verifyBalance.mjs`，检查 20 关目标、球速、奖励、完整升级经济和实体上限。
- 新增 `verifyPerformanceBudgets.mjs`，压力验证三档粒子、拖尾、冲击波与浮分硬上限。
- 新增视觉设置、跨档玩法等价、v3 迁移、存储降级和设置持久化测试，Node 测试扩充到 55 项。
- 新增 [v0.9.0 验收报告](./docs/v0.9.0验收报告.md)。

### 调整

- 低档关闭高频实体辉光并降低粒子密度，但保留砖块耐久点、熔芯符号、危险轮廓和招架提示。
- Canvas 瞬态对象在切换到更低档时立即裁剪，运行文本状态新增档位、预算、震屏与减弱闪光字段。
- 页面加入安全区变量、`viewport-fit=cover`、动态视口高度与传统 `vh` 回退、触摸行为和键盘焦点样式。
- 窄屏顶部导航适配四个入口；设置页在手机端纵向滚动，游戏页在 320×568 仍保持单屏。
- 同屏掉落上限保持 28，不随视觉档位变化；性能优化只作用于纯视觉对象，避免改变终局资源供给。
- 页面标题、构建标识、页脚和包版本更新到 v0.9.0。

### 验证

- `pnpm test` 55/55 通过；`pnpm test:balance` 与 `pnpm test:performance` 均无失败项。
- 第一、二、三、四章共 20 关分组真实物理回归全部胜利；第 20 关经过四阶段、两座模块全毁并以 1 条生命完成。
- 无尽模式连续清除 5 波并进入第 6 波；生产构建和差异检查通过。
- 官方网页游戏客户端完成低档熔芯/三耐久砖视觉与预算检查。
- 用户当前 Chrome 完成 1300×803、1280×720、390×844、360×800、320×568 设置页与 Boss 实战验收，控制台 0 警告/错误。
- v2→v3 后用户 151 晶币、2 星、14471 最高分、29 最佳连击和关卡解锁保持不变；原画质、震屏、闪光设置已恢复并刷新确认。
