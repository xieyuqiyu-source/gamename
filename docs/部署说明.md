# 霓虹破界部署说明

本文档适用于 `v1.0.0` 及后续纯前端版本。游戏没有服务器接口、数据库或运行时密钥，生产构建产物可以部署到任意支持静态文件的服务。

## 1. 环境要求

- Node.js `20.19+` 或 `22.12+`
- pnpm `10+`
- 支持 Canvas 2D、ES Modules 和 localStorage 的现代浏览器

## 2. 生产构建

```bash
pnpm install --frozen-lockfile
pnpm exec playwright install chromium
pnpm test:release
```

`test:release` 会依次执行目标范围审计、55 项以上单元测试、平衡/性能专项、20 关真实物理回归、无尽五波、生产构建和生产包浏览器门禁。最后一项会把 `dist/` 挂载到 `/games/neon-breaker/` 子目录，真实检查相对资源、390/320 像素布局、四张画质卡片和运行时错误。

只执行构建：

```bash
pnpm build
```

本地检查生产包：

```bash
pnpm preview --host 0.0.0.0
```

默认地址为 `http://localhost:4173/`。

## 3. 部署产物

只需要上传 `dist/` 的全部内容。不要上传 `src/`、`tests/`、`node_modules/` 或开发脚本。

Vite 使用相对基础路径 `./`，因此构建产物既可放在域名根目录，也可放在 `/games/neon-breaker/` 等子目录，不需要重新构建。

生产包包含：

- 入口 HTML 与哈希化 CSS/JavaScript
- 正式 SVG 图标
- Web App Manifest
- `.nojekyll` 静态托管标记

## 4. 常见平台

### 4.1 GitHub Pages

1. 执行 `pnpm build`。
2. 将 `dist/` 发布到 Pages 使用的分支或 Actions artifact。
3. 仓库 Pages 设置选择对应来源。
4. 访问发布地址，检查标题、图标、Canvas 和存档刷新恢复。

相对基础路径已经兼容仓库子路径，不需要把仓库名写死在配置中。

### 4.2 Netlify / Cloudflare Pages / Vercel

| 配置 | 值 |
| --- | --- |
| 安装命令 | `pnpm install --frozen-lockfile` |
| 构建命令 | `pnpm build` |
| 输出目录 | `dist` |

游戏没有前端路由，通常不需要 SPA 回退规则。

### 4.3 Nginx

```nginx
server {
  listen 80;
  server_name example.com;
  root /var/www/neon-breaker/dist;
  index index.html;

  location / {
    try_files $uri $uri/ =404;
  }

  location ~* \.(js|css|svg|png|webmanifest)$ {
    expires 7d;
    add_header Cache-Control "public, max-age=604800, immutable";
  }
}
```

如果频繁更新版本，可把 `index.html` 的缓存时间设为 0，哈希资源继续长期缓存。

## 5. 存档与域名

正式档案保存于当前站点来源的 localStorage，键为 `gamename:save`。以下变化会形成不同的存档空间：

- HTTP 与 HTTPS 切换
- 域名或端口变化
- 浏览器无痕模式或清理站点数据

发布后应保持正式域名和路径稳定。路径变化通常不会分离 localStorage，但域名变化会。浏览器拒绝本地存储时，游戏会切换到临时内存并在设置页显示 `VOLATILE`。

当前生产环境固定为：

- 生产地址：`https://game.ccoos.cn/neon-breaker/`
- 服务器部署目录：`/srv/game.ccoos.cn/neon-breaker`
- 服务器部署脚本：`/usr/local/bin/deploy-neon-breaker`
- Nginx 路径：`/neon-breaker/`

部署由 GitHub Actions 触发，但构建和产物切换都在服务器上完成。

## 6. 发布后检查

- 页面标题显示“霓虹破界：Neon Breaker”。
- 浏览器标签使用正式霓虹核心图标。
- 桌面和手机均能进入战役、强化、设置和游戏。
- 发球、移动、暂停、恢复和全屏工作正常。
- 刷新后晶币、关卡记录和设置可以恢复。
- 控制台没有错误或警告。
- 390×844 与 360×800 没有横向溢出，底部主操作完整可见。

## 7. 回滚

每个正式阶段都有 Git 标签。需要回滚时重新构建目标标签并替换静态产物：

```bash
git switch --detach v1.0.0
pnpm install --frozen-lockfile
pnpm build
```

不要清理浏览器 localStorage；SAVE V3 档案与 `v1.0.0` 兼容。
