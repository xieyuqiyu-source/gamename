# Neon Breaker v0.9.0 验收报告

验收日期：2026-07-20  
阶段名称：平衡、性能与兼容性  
版本号：`v0.9.0`  
预发布里程碑：`9 / 9`，`100%`

## 1. 阶段结论

`v0.9.0` 完成发行前的数值、特效性能、双端布局和存档异常收口。游戏新增自动/高/中/低画面选择、真实可切换的震屏与减弱闪光、设备与存档诊断页面，并将正式存档提升到 v3。

视觉档位只控制粒子、拖尾、冲击波、浮分和辉光，不改变固定时间步、碰撞、球速、得分、掉落或 Boss 规则。20 关、4 个 Boss、无尽模式和 7 类永久升级继续使用同一套玩法数据。

## 2. 交付范围

### 2.1 视觉性能分档

| 档位 | 粒子 | 拖尾 | 冲击波 | 浮分 | 高频辉光 |
| --- | ---: | ---: | ---: | ---: | --- |
| HIGH | 600 | 150 | 28 | 36 | 开启 |
| MEDIUM | 360 | 96 | 20 | 28 | 开启 |
| LOW | 180 | 56 | 12 | 20 | 关闭 |

- `AUTO` 根据系统减少动态效果偏好、视口宽度、硬件线程与设备内存解析到三档之一。
- 运行中切换档位会立刻裁剪超过新预算的瞬态对象，不需要重新开始关卡。
- 低档仍保留普通/强化/三耐久/熔芯砖的颜色、边框、耐久点和机关符号。
- 震屏关闭后逻辑震动值不参与绘制；减弱闪光把全屏白闪限制在 12% 以内。

### 2.2 设置与兼容性页面

- 全局导航新增“设置”，独立展示当前解析档位和各类对象预算。
- 支持自动、高、中、低四种选择，修改后立即保存。
- 支持画面震动和减弱闪光开关。
- 展示线程、设备内存、系统动态偏好和正式存档版本。
- 本地存储不可用时显示 `VOLATILE` 与临时内存提示。
- 页面加入 `viewport-fit=cover`、`env(safe-area-inset-*)`、`100dvh` 与 `100vh` 回退。
- 键盘焦点、触摸行为和系统减少动态效果媒体查询均有明确样式。

### 2.3 正式存档 v3

新增字段：

```js
settings: {
  effectQuality: 'auto',
  screenShake: true,
  reducedFlash: false,
  controlMode: 'auto',
}
```

- v1/v2 正式档案会升级到 v3，并保留晶币、关卡记录、解锁、升级、无尽纪录与原有高/低设置。
- 损坏 JSON 继续先备份原文，再建立安全档案。
- localStorage 获取、读取或写入抛错时切换到会话内存，不中断启动。
- 设置页会明确区分浏览器本地存储与临时内存模式。

## 3. 数值与经济审计

`pnpm test:balance` 对全部配置执行以下断言：

- 每章 5 关，章节内目标分数、目标连击、清关奖励和球速不倒退。
- 四章基础球速从 `1.000` 递增到 `1.340`。
- Boss 关奖励高于同章普通关。
- 同屏小球上限 12、掉落物上限 28。
- 20 关固定清关奖励共 1568 晶币，累计星级奖励共 1175 晶币。
- 不计算随机掉落时，保证收入 2743 晶币；全部升级成本 2720，余量 23。
- 第一章保证收入 508，可覆盖全部七项第一级升级的 350 晶币成本。

性能收口初期曾把掉落物上限从 28 压到 24，真实物理回归显示第 20 关可能在 Boss 最后 1 点耐久时失败。最终恢复 28，把档位差异严格限制在视觉对象，终局回归重新通过。
