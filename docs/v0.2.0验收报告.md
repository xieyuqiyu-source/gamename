## 3. 完整游戏回归

使用自动挡板跟随球体并加入周期性偏移，运行完整第 1 关：

```json
{
  "frames": 6618,
  "mode": "won",
  "lives": 3,
  "score": 10260,
  "remaining": 0
}
```

结果证明关卡不依赖调试入口即可正常清空全部砖块并触发胜利。

## 4. UI 与画面回归

### 4.1 标准桌面

- 三栏结构清晰：任务、竖屏战场、操作终端。
- 战场高度受视口约束，页面首屏可显示完整操作区。
- 当前得分、最高得分、砖块进度和生命状态一致。
- 普通砖使用青绿色，强化砖使用紫色并带金色耐久点。

### 4.2 1280×720 小型桌面

- 完整页面在单屏内显示。
- 两侧面板与中央战场无重叠。
- 按钮、键位说明和底部版本信息没有换行或截断。

### 4.3 390×844 标准手机

- 隐藏桌面侧栏，保留标题、战场、移动状态和主操作按钮。
- Canvas 占据主要视口，底部按钮可直接触摸。
- 生命、状态和砖块数量在战场外形成简洁摘要。

### 4.4 360×800 窄屏手机

- 标题、战场和主按钮仍能完整显示。
- HUD 无遮挡，砖块与强化砖耐久点清晰可读。
- 无水平滚动和文本截断。

### 4.5 状态画面

- 菜单：品牌、章节、标题和主按钮层级明确。
- 准备：发球说明位于空白战场中部，不遮挡挡板与球。
- 游戏：减少覆盖文字，只保留 HUD、砖块、球和挡板。
- 暂停：背景压暗，当前球和砖块位置仍可辨认。
- 胜利：金色 `CLEAR!` 与再次挑战按钮。
- 失败：粉红 `ENERGY LOST` 与重新挑战按钮。
