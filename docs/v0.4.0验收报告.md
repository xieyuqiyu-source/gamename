## 2. 完整游戏流程

### 2.1 页面状态

| 状态 | 玩家操作 | 下一状态 | 验收 |
| --- | --- | --- | --- |
| 标题 `menu` | 点击“进入任务”或 Enter | 任务简报 | 通过 |
| 任务简报 `briefing` | 开始挑战 / 返回标题 | 准备发球 / 标题 | 通过 |
| 准备 `ready` | 移动挡板并发球 | 游戏中 | 通过 |
| 游戏 `playing` | 移动、激光、暂停 | 暂停 / 胜负 | 通过 |
| 暂停 `paused` | 点击、Space 或 P | 3 秒倒计时 | 通过 |
| 倒计时 `countdown` | 等待 / 再次点击取消 | 游戏中 / 暂停 | 通过 |
| 胜利 `won` | 再次挑战 / 返回标题 | 准备 / 标题 | 通过 |
| 失败 `lost` | 重新挑战 / 返回标题 | 准备 / 标题 | 通过 |

每次正式开始挑战都会生成递增的 `runId`。胜负结算使用 `runId` 做当前页面内的幂等保护，重复状态发布不会重复增加挑战次数或奖励。

### 2.2 第一关三星规则

| 星级条件 | 参数 | 结果 |
| --- | --- | --- |
| 第一星 | 清除全部 51 块砖 | 通过 |
| 第二星 | 通关时至少剩余 2 条生命 | 通过 |
| 第三星 | 15000 分或 35 连击 | 通过 |

三星记录采用历史最高值。后续低星通关或失败不会降低既有星级、最高分和最佳连击。

### 2.3 结算内容

- 最终得分。
- 本局最高连击。
- 本局获得晶币。
- 剩余生命。
- 0～3 星评价。
- 清关固定奖励 20 晶币。
- 正式存档同步状态。
- 再次挑战与返回标题。

## 3. 正式存档验收

### 3.1 存储键与结构

正式键：`gamename:save`  
当前版本：`saveVersion = 1`

持久状态分为：

- `profile`：创建和更新时间、迁移来源。
- `currency`：晶币。
- `campaign`：最高解锁关卡、关卡记录、最近结算。
- `endless`：无尽模式解锁和历史占位。
- `upgrades`：7 类永久升级占位。
- `settings`：特效质量、震屏和控制模式。

第 1 关记录包含：

- 历史最高星级。
- 历史最高分。
- 历史最佳连击。
- 挑战次数和通关次数。
- 最佳剩余生命。
- 最后游玩时间。

### 3.2 存档场景

| 场景 | 预期 | 验收 |
| --- | --- | --- |
| 首次进入 | 建立默认正式存档 | 通过 |
| 胜利结算 | 保存晶币、星级、分数、连击和通关 | 通过 |
| 失败结算 | 保留本局晶币、增加挑战，不降低历史记录 | 通过 |
| 刷新页面 | 恢复晶币、星级、最高分和最佳连击 | 通过 |
| 旧快照迁移 | 迁移 coins、bestScore、maxCombo 和设置 | 通过 |
| 迁移备份 | 保存 `gamename:game:migrated` 原始字符串 | 通过 |
| 损坏正式存档 | 备份原文并建立新档 | 通过 |
| 未知字段 | 忽略并归一化已知结构 | 通过 |
| 重置进度 | 第一次解锁确认，第二次清空 | 通过 |

迁移实测使用 77 晶币、12345 最高分、29 连击、低特效和关闭震屏的旧快照；全部字段进入正式存档，旧键被移除，原始字符串保留在迁移备份键。

损坏恢复实测将正式键写入无效 JSON。加载后原文保存在带时间戳的 `gamename:save:corrupt:*`，主键恢复为可解析的默认 v1 存档。

自动挡板只控制挡板目标位置，并在激光有效时主动射击；没有直接修改砖块耐久。

| 固定时间步帧数 | 3739 |
| 模拟时间 | 31.2 秒 |
| 最终模式 | won |
| 剩余砖块 | 0 / 51 |
| 最终生命 | 3 |
| 最终得分 | 22727 |
| 最高连击 | 60 |
| 砖块掉落晶币 | 41 |
| 清关固定奖励 | 20 |

结算后等待 0.75 秒，粒子、波纹、震屏全部自然收束；活动球、掉落物、弹体和即时模块均清空。

## 5. 交互回归

通过外部 Vue 按钮和 Canvas 输入分别验证：

- 标题进入任务简报。
- 简报返回标题。
- 简报开始挑战。
- 准备状态发球。
- 游戏中暂停。
- 暂停后恢复。
- 暂停 1 秒后挡板位置、粒子数量和道具剩余时间均保持不变。
- 页面失焦自动暂停，确认后经过 3 秒倒计时再恢复。
- 胜利后重试生成新的运行编号。
- 胜利/失败结算返回标题。
- 两步确认重置，第一次不改变存档，第二次恢复默认值。

状态序列实测：

```text
menu → briefing → menu → briefing → ready(runId=1)
→ playing → paused → countdown → playing → won
→ ready(runId=2) → won → menu
```

开发环境新增 `window.__NEON_BREAKER_SAVE_TEST__`：

- `summary()`：读取可断言存档摘要。
- `export()`：导出正式存档对象。
- `reload()`：重新加载正式存档。
- `reset()`：建立新档并同步引擎。
